Radio\+Lib was originally created as a driver for \href{https://github.com/jgromes/RadioShield}{\texttt{ {\bfseries{Radio\+Shield}}}}, but it can be used to control as many different wireless modules as you like -\/ or at least as many as your Arduino can handle!\hypertarget{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md61}{}\doxysubsection{Supported modules\+:}\label{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md61}

\hypertarget{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md66}{}\doxysubsection{Where should I start?}\label{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md66}
First of all, take a look at the \href{https://github.com/jgromes/RadioLib/tree/master/examples}{\texttt{ examples}} and the \href{https://github.com/jgromes/RadioLib/wiki}{\texttt{ Wiki}} -\/ especially the \href{https://github.com/jgromes/RadioLib/wiki/Basics}{\texttt{ Basics}} page. There\textquotesingle{}s a lot of useful information over there. Also, you should check out \href{https://github.com/jgromes/RadioShield}{\texttt{ Radio\+Shield}} -\/ open source Arduino shield that will allow you to easily connect any two wireless modules supported by Radio\+Lib!\hypertarget{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md67}{}\doxysubsection{Help, my module isn\textquotesingle{}t working!}\label{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md67}
The fastest way to get help is by creating an \href{https://github.com/jgromes/RadioLib/issues/new?assignees=&labels=&template=bug_report.md&title=}{\texttt{ issue}} using the appropriate template. It is also highly recommended to try running the examples first -\/ their functionality is tested from time to time and they should work. Finally, Radio\+Lib is still under development, which means that sometimes, backwards-\/incompatible changes might be introduced. Though these are kept at minimum, sometimes it is unavoidable. You can check the \href{https://github.com/jgromes/RadioLib/releases}{\texttt{ release changelog}} to find out if there\textquotesingle{}s been such a major change recently.\hypertarget{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md68}{}\doxysubsection{Radio\+Lib doesn\textquotesingle{}t support my module! What should I do?}\label{md__libraries__radio_lib__r_e_a_d_m_e_autotoc_md68}
Start by creating new issue (if it doesn\textquotesingle{}t exist yet). If you have some experience with Arduino and C/\+C++ in general, you can try to add the support yourself! Use the template files in {\ttfamily /extras/} folder to get started. This is by far the fastest way to implement new modules into Radio\+Lib, since I can\textquotesingle{}t be working on everything all the time. If you don\textquotesingle{}t trust your programming skills enough to have a go at it yourself, don\textquotesingle{}t worry. I will try to implement all requested modules, but it will take me a while.
